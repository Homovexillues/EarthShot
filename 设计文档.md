# EarthShot 设计文档

> 一个轻量、内存内完成、可扩展的 Avalonia 截图工具。灵感来自 Flameshot，但只做一件事做到位：**框选即识别 QR 码**。

---

## 1. 项目目标

### 1.1 MVP（v0.1）必须做到

- 全局快捷键（如 `Ctrl+Shift+X`）唤起全屏透明覆盖层
- 鼠标拖拽框选矩形区域
- **松开鼠标立即识别**选中区域内的 QR 码
- 识别到 → 内容复制到剪贴板 + 屏幕简短提示（2 秒消失）
- 未识别 → 提示"未找到 QR 码"
- 按 `Esc` 或右键取消
- **像素数据全程在内存中流转，不写盘**

### 1.2 非目标（MVP 不做）

- 保存图片到文件
- 标注（箭头、文字、涂鸦）
- OCR 文字识别
- 多语言 UI
- 多平台（先 Windows）

### 1.3 设计约束

- **扩展不改核心**：加新功能（OCR、取色、上传图床）应该是"加一个实现 + 注册"，不改管道
- **可测试**：核心逻辑（捕获、处理、动作）headless 可运行，不依赖 UI
- **AOT-friendly**：v0.1 不强制 AOT，但不使用会被 AOT 拒绝的模式
- **克制**：保持最小化，宁可等需求出现再加抽象，不做"为未来准备"的脚手架

## 2. 核心架构：三段管道

整个工具抽象为一条三节流水线：

```
┌──────────┐    ┌─────────────┐    ┌──────────┐
│ Capturer │ ─► │ Processors  │ ─► │ Actions  │
└──────────┘    └─────────────┘    └──────────┘
   捕获像素      识别/分析图像        处理结果
```

**三段的意义**：

- **Capturer**：决定"从哪取像素"（屏幕区域 / 剪贴板 / 文件）
- **Processor**：决定"对像素做什么"（识别 QR / OCR / 取色 / 测距）
- **Action**：决定"结果怎么用"（进剪贴板 / 打开浏览器 / 显示通知）

三段通过**最小稳定接口**连接，任一段换实现都不影响另两段。

## 3. 核心抽象

### 3.1 数据类型（尽量少，尽量稳定）

```csharp
// 捕获到的原始图像 —— 内存中的像素
public readonly record struct CapturedImage(
    ReadOnlyMemory<byte> Pixels,  // RGBA 或 BGRA 字节流
    int Width,
    int Height,
    int Stride,
    PixelFormat Format
);

// 处理器的产出 —— 一次处理可能零到多个结果
public abstract record ProcessingResult;
public record QrCodeResult(string Text, RectInImage Bounds) : ProcessingResult;
public record OcrResult(string Text, ...) : ProcessingResult;  // 未来
public record ColorPickedResult(Rgba32 Color) : ProcessingResult;  // 未来

// 动作的输入上下文
public record ActionContext(CapturedImage Source, IReadOnlyList<ProcessingResult> Results);
```

### 3.2 三个核心接口

```csharp
public interface IScreenCapturer
{
    CapturedImage Capture(PixelRect region);
}

public interface IImageProcessor
{
    string Name { get; }
    IEnumerable<ProcessingResult> Process(CapturedImage image);
}

public interface IResultAction
{
    // 询问"我能处理这种结果吗？"
    bool CanHandle(ProcessingResult result);
    // 执行（可能异步）
    Task ExecuteAsync(ProcessingResult result, ActionContext context);
}
```

**关键约定**：

- `IImageProcessor.Process` 不抛异常，通过返回空集合表达"没识别到"
- `IResultAction` 互相独立，同一个 result 可能触发多个 action（如"复制到剪贴板"和"显示通知"）
- 处理器和动作的执行顺序由管道调度，不由实现者互相依赖

### 3.3 管道调度

```csharp
public sealed class ScreenshotPipeline
{
    public ScreenshotPipeline(
        IScreenCapturer capturer,
        IEnumerable<IImageProcessor> processors,
        IEnumerable<IResultAction> actions);

    public async Task RunAsync(PixelRect region, CancellationToken ct);
}
```

MVP 阶段管道就这么朴素——不做异步流水线、不做并行、不做取消优化。等性能真成为问题再升级。

## 4. UI 层：覆盖窗口

### 4.1 窗口设计

单个 **`OverlayWindow`**，特征：

- 全屏（覆盖虚拟屏幕总 bounds，处理多显示器）
- 透明背景 + 半透明遮罩（`SystemDecorations="None"`, `TransparentLevelHint="Transparent"`, `Background="Transparent"`）
- TopMost
- 无任务栏图标
- 捕获所有鼠标和键盘输入

### 4.2 交互状态机

```
Idle (等待按下鼠标左键)
  │
  ▼  按下鼠标
Selecting (拖拽中，实时绘制选区矩形)
  │
  ▼  松开鼠标
Processing (调用管道，显示转圈或直接完成)
  │
  ▼  管道结束
Feedback (2 秒提示条) ─► 关闭窗口
  │
  ▼  按 Esc / 右键 （任何状态）
Cancelled ─► 关闭窗口
```

### 4.3 多显示器 & DPI

已知难点。处理原则：

- 唤起时查询所有屏幕的**物理 bounds**（非逻辑），窗口覆盖 `bounds.Union()`
- 捕获时把 Avalonia 的逻辑坐标转成物理像素（乘以 `RenderScaling`）
- 截图所得像素的坐标系和鼠标拖出的 Rect 必须对得上——开发时先单屏跑通，再补多屏测试

## 5. 项目结构

**原则**：不先拆成 5 个项目。先用单项目 + 清晰目录。等某部分真需要独立打包了再拆。

```
EarthShot/
├── EarthShot.sln
├── src/
│   ├── EarthShot.Core/                # 核心库（headless，无 UI 依赖）
│   │   ├── Capturing/
│   │   │   ├── IScreenCapturer.cs
│   │   │   ├── CapturedImage.cs
│   │   │   └── Windows/
│   │   │       └── GdiScreenCapturer.cs
│   │   ├── Processing/
│   │   │   ├── IImageProcessor.cs
│   │   │   ├── ProcessingResult.cs
│   │   │   └── QrCodeProcessor.cs     # ZXing.Net 实现
│   │   ├── Actions/
│   │   │   ├── IResultAction.cs
│   │   │   └── ClipboardAction.cs
│   │   ├── Pipeline/
│   │   │   └── ScreenshotPipeline.cs
│   │   └── EarthShot.Core.csproj
│   │
│   └── EarthShot.App/                 # Avalonia UI
│       ├── Views/
│       │   ├── OverlayWindow.axaml
│       │   └── ToastView.axaml
│       ├── ViewModels/
│       │   └── OverlayViewModel.cs
│       ├── Services/
│       │   ├── HotkeyService.cs       # Win32 RegisterHotKey
│       │   └── TrayService.cs
│       ├── App.axaml
│       ├── Program.cs
│       └── EarthShot.App.csproj
│
└── tests/
    └── EarthShot.Core.Tests/
```

**为什么 Core 和 App 分开**：
- Core 可以 headless 跑单元测试（不启动 Avalonia）
- 未来若做 CLI 版本、被 Rider 插件调用、作为库发布，都不用动
- 这是性价比最高的一次分包——两个项目，不多不少

**不做的事**：不做 `EarthShot.Platform.Windows`、`EarthShot.Processing.Qr` 这类细粒度拆包。等到确实要发 NuGet 或剥离单元再说。

## 6. 技术选型

| 领域 | 选择 | 备注 |
|------|------|------|
| UI 框架 | **Avalonia 11.x** | CompiledBindings 全程启用 |
| MVVM | **CommunityToolkit.Mvvm** | 用它的 Source Generator，AOT-friendly |
| 屏幕捕获 | **GDI BitBlt**（P/Invoke）| 简单、跨 Win 版本稳定。DXGI Desktop Duplication 更快但 MVP 用不上 |
| QR 识别 | **ZXing.Net**（0.16+）| 成熟稳定，AOT 需验证但主流报告可行 |
| 图像处理 | **SkiaSharp** 或裸字节 | 先裸字节 `ReadOnlyMemory<byte>`，不引入 SkiaSharp 除非真需要 |
| 全局快捷键 | Win32 `RegisterHotKey` P/Invoke | 无需三方库 |
| 剪贴板 | Avalonia `IClipboard` | 图像和文本都有 API |
| 日志 | `Microsoft.Extensions.Logging` + 简单 FileSink | 或直接 Serilog，不要再自写 Logger 了（见前讨论） |
| 配置 | `System.Text.Json` + Source Generator | `%APPDATA%/EarthShot/settings.json` |
| DI | `Microsoft.Extensions.DependencyInjection` | 标准选择 |
| 打包 | 单文件 self-contained → AOT | 见下节 |

## 7. AOT 计划

### 分阶段

- **v0.1**：不启用 AOT。但代码坚持 AOT-friendly：
  - XAML 全部 `x:CompileBindings="True"`
  - `[JsonSerializable]` 标注所有序列化类型
  - 零反射（少数反射点集中化，便于后续替换）
- **v0.2**：本地试跑 `dotnet publish -c Release -r win-x64 --self-contained`，观察启动速度和体积
- **v0.3**：启用 `<PublishAot>true</PublishAot>`，处理 ILC 警告。ZXing.Net 如有坑，查社区方案或临时换库

### 预期风险

- 部分 ZXing.Net 版本依赖 `System.Drawing.Common`（在 Linux AOT 会炸）
- 反射绑定在 Avalonia XAML 里极易漏网——必须开启 `AvaloniaUseCompiledBindingsByDefault=true`
- P/Invoke 签名错误在 AOT 下直接崩而非抛异常——要谨慎

## 8. MVP 实施路线（按 1 周/晚 2 小时估算）

| 里程碑 | 目标 | 大致工时 |
|--------|------|----------|
| **M1 骨架搭起** | 空白 Avalonia + 单项目 + 全屏透明覆盖窗口 | 1 晚 |
| **M2 选区交互** | 能拖出矩形选框、按 Esc 退出 | 1 晚 |
| **M3 屏幕捕获** | 松开鼠标时拿到 `CapturedImage`，打印尺寸 | 1-2 晚 |
| **M4 QR 识别** | 集成 ZXing，显示识别结果 | 1 晚 |

## 9. 扩展路径（v0.2+，按需实现，不预先编码）

每一条都只需要"加文件 + 注册服务"，不动核心：

| 新功能 | 做法 |
|--------|------|
| **OCR 文字识别** | 新增 `TesseractOcrProcessor : IImageProcessor`；结果类型 `OcrResult`；默认 action 是复制文字 |
| **取色器** | 新增 `ColorPickProcessor`（1x1 捕获后读像素）；`ColorPickedResult`；action 复制 HEX |
| **保存到文件** | 新增 `SaveToFileAction` 实现 `IResultAction` |
| **上传图床** | 新增 `UploadAction`（对 `FullImageResult` 生效） |
| **标注功能** | 在 UI 层插入"编辑模式" ——这个会动 UI 状态机，算中等改动，不算无侵入。设计时就考虑好：管道在"捕获后、处理前"预留一个可选的编辑阶段 |
| **截取特定窗口** | 新增 `WindowCapturer : IScreenCapturer`，绑定不同触发源 |

**一条原则**：**凡是能通过"新增实现类 + DI 注册"完成的，都不许改 Core**。反例：如果有人想加"标注", 要改 OverlayWindow 的状态机——这说明状态机接口不够稳定，到时再讨论是否要为它单独出一个扩展点。

## 10. 已知风险 / 需要研究的点

1. **Avalonia 11 全屏透明窗口在 Windows 上的具体实现**：需要验证 `Background=Transparent + BackgroundMaterial=None + WindowState=FullScreen` 的组合是否真的能做到点击穿透的反面（即：能接收输入但看穿背景）。可能要 Win32 `SetWindowLong` 调整 `WS_EX_LAYERED`。
2. **多显示器 DPI 混合缩放**：Windows 允许每块屏不同 DPI。截图坐标必须以物理像素为准。
3. **ZXing.Net AOT 兼容性**：最新版本 release notes 要查，必要时换 [ZXing.Net.Bindings.ImageSharp](https://github.com/micjahn/ZXing.Net)。
4. **全局快捷键和其他软件冲突**：建议默认用一个冷门组合（`Ctrl+Shift+Alt+Q`），并允许配置。
5. **拒绝权限下启动**：Windows 某些 UAC 配置下 `RegisterHotKey` 可能失败。要有 fallback（仅托盘点击唤起）。

## 11. 暂不考虑但要记一笔的事

- **插件 DLL 热加载**：不做。如果真到那天再说，参照 `IAssemblyLoadContext` 而不是自造 FiTools 式容器。
- **跨平台**：MVP Windows-only。Linux/macOS 的屏幕捕获是另一套 API（XShm、AVFoundation），留到有需求再做。
- **配置 UI**：MVP 不做。配置文件手改。
- **自动更新**：不做。用 winget 或手动下载。
- **遥测**：不做。
